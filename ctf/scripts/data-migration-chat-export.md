# ChargingTheFuture Data Migration Chat Export

## Summary
This chat details the process for migrating all data from the legacy platform database (migration-test) to the new rewrite-test database using copy-paste SQL, due to Neon restrictions on dblink. The approach is to generate SQL INSERT statements for each table in migration-test, then paste them into rewrite-test after applying the new schema.

## Key Steps
1. Apply the new schema (ctf/schema.sql) to rewrite-test.
2. For each table, run a SQL generator query in migration-test to output INSERT statements.
3. Copy the output and paste it into rewrite-test.
4. Repeat for all tables.
